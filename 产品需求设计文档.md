# Atoms-Demo 产品需求设计文档

## 1. 文档概述

### 1.1 产品背景
Atoms（https://atoms.dev/）是下一代 AI Agent 平台。本 Demo 旨在实现一个可运行的网页应用，具备类似 Atoms 的能力与 UI 交互体验，即通过智能体驱动的方式完成代码（应用）生成，并将生成的应用以可视化网页形式进行展示。

### 1.2 产品目标
- 构建一个可运行的 AI 代码生成平台 Demo
- 实现用户通过自然语言描述需求，系统自动生成可运行的网页应用
- 具备真实交互能力和数据持久化能力

### 1.3 评估维度
- 完成度：功能完整性、稳定性与基本工程质量
- 工程思维：任务拆解、技术选型、复杂度控制与取舍能力
- 用户体验：交互是否清晰、流程是否顺畅、是否"可用"
- 创新性：是否有亮点设计、独特视角或可扩展潜力
- 可交付性：文档清晰度、可运行程度、整体完成质量

## 2. 功能需求

### 2.1 核心功能 (P0)

#### 2.1.1 聊天输入区
| 项目 | 描述 |
|------|------|
| 功能描述 | 用户通过输入框输入自然语言需求描述 |
| 交互要求 | 支持多行输入、发送按钮、回车发送、加载状态提示 |
| 输入限制 | 最大 2000 字符 |
| 异常处理 | 空输入提示、网络错误提示 |

#### 2.1.2 代码生成与预览区
| 项目 | 描述 |
|------|------|
| 功能描述 | 展示 AI 生成的代码，支持语法高亮 |
| 展示要求 | 分标签页展示 HTML/CSS/JS 代码 |
| 交互要求 | 支持代码复制、支持手动编辑 |
| 状态展示 | 生成中动画、生成成功/失败状态 |

#### 2.1.3 应用预览区
| 项目 | 描述 |
|------|------|
| 功能描述 | 实时渲染生成的网页应用 |
| 技术要求 | 使用 iframe 沙箱隔离，确保安全 |
| 交互要求 | 支持刷新预览、全屏预览 |
| 响应式 | 支持不同尺寸预览（桌面/平板/手机） |

#### 2.1.4 数据持久化
| 项目 | 描述 |
|------|------|
| 功能描述 | 保存生成的项目代码和用户历史 |
| 存储内容 | 项目名称、代码内容、创建时间、更新时间 |
| 技术方案 | 后端 Python + SQLite 数据库 |

### 2.2 延伸功能 (P1)

#### 2.2.1 历史记录
| 项目 | 描述 |
|------|------|
| 功能描述 | 查看和切换历史生成记录 |
| 交互要求 | 列表展示、缩略图预览、点击切换 |
| 操作支持 | 删除历史记录、重命名项目 |

#### 2.2.2 代码迭代生成
| 项目 | 描述 |
|------|------|
| 功能描述 | 基于已有代码进行二次修改 |
| 交互要求 | 支持选择历史版本、对比差异 |
| AI 能力 | 理解上下文，进行增量修改 |

#### 2.2.3 项目导出
| 项目 | 描述 |
|------|------|
| 功能描述 | 导出生成的代码为独立文件 |
| 导出格式 | ZIP 压缩包（包含 index.html） |
| 扩展能力 | 一键部署到静态托管平台 |

## 3. 技术架构

### 3.1 技术栈选型

| 层级 | 技术 | 说明 |
|------|------|------|
| **后端框架** | Python + FastAPI | 轻量、高性能、异步支持 |
| **依赖管理** | uv | 快速 Python 包管理 |
| **数据库** | SQLite + SQLAlchemy | 轻量级，零配置 |
| **前端框架** | Vue 3 + TypeScript | 渐进式框架，组合式 API |
| **构建工具** | Vite | 快速开发和热更新 |
| **UI 组件库** | Element Plus / Naive UI | 完整的组件生态 |
| **代码编辑器** | Monaco Editor | VS Code 同款编辑器 |
| **状态管理** | Pinia | Vue 3 官方推荐状态管理 |
| **HTTP 客户端** | Axios | 请求库 |
| **AI 调用** | Atoms API / Claude API | 代码生成能力 |
| **部署方案** | Docker + 任意云平台 | 容器化部署 |

## 4. API 接口设计

### 4.1 接口列表

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | /api/projects | 创建新项目 |
| GET | /api/projects | 获取项目列表 |
| GET | /api/projects/{id} | 获取项目详情 |
| PUT | /api/projects/{id} | 更新项目 |
| DELETE | /api/projects/{id} | 删除项目 |
| POST | /api/generate | 调用 AI 生成代码 |
| POST | /api/sessions | 创建新会话 |
| GET | /api/sessions/{id}/messages | 获取会话消息 |

### 4.2 核心接口定义

#### POST /api/generate
**请求体:**
```json
{
  "prompt": "创建一个待办事项应用",
  "project_id": 1,
  "context": "之前的代码..."
}
```

**响应:**
```json
{
  "success": true,
  "data": {
    "html": "<!DOCTYPE html>...",
    "css": "body { ... }",
    "js": "console.log('hello')"
  },
  "message": "生成成功"
}
```

#### GET /api/projects
**响应:**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "待办事项应用",
      "created_at": "2026-04-01T10:00:00Z",
      "updated_at": "2026-04-01T10:30:00Z"
    }
  ]
}
```

## 7. 开发计划

### 7.1 时间分配 (总计 6-8 小时)

| 阶段 | 时间 | 任务 | 交付物 |
|------|------|------|--------|
| 第一阶段 | 1h | 环境搭建、项目初始化 | 可运行的前后端框架 |
| 第二阶段 | 1.5h | 后端 API 开发 | 完整的 REST API |
| 第三阶段 | 2h | 前端核心功能 | 聊天 + 代码预览 + 应用预览 |
| 第四阶段 | 1h | AI 集成 | 可生成代码的完整流程 |
| 第五阶段 | 1h | 数据持久化 | 项目保存和历史记录 |
| 第六阶段 | 0.5h | 延伸功能 | 导出/分享等亮点功能 |
| 第七阶段 | 1h | 测试优化 + 文档 | 可交付的 Demo |

### 7.2 里程碑

- **M1 (2.5h)**: 基础框架完成，前后端可通信
- **M2 (4.5h)**: 核心功能完成，可生成并预览应用
- **M3 (6h)**: 全部功能完成，可完整演示
- **M4 (7h)**: 测试优化，文档完善

## 8. 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| AI API 调用失败 | 中 | 高 | 准备 Mock 数据，支持降级展示 |
| 前端构建问题 | 低 | 中 | 使用成熟模板，减少自定义配置 |
| 时间不足 | 中 | 中 | 优先保证 P0 功能，P1 功能酌情裁剪 |
| iframe 安全限制 | 低 | 低 | 使用 srcdoc 方式渲染，避免跨域问题 |
